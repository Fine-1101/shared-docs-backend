# 在线共享文档前端开发项目计划书

## 一、项目概述

### 1.1 项目名称
Shared Docs Platform - 在线协作编辑平台前端

### 1.2 项目背景
基于现有的 Spring Boot 后端服务，开发一个支持实时协作编辑的在线文档平台前端应用。该平台允许多用户同时编辑同一文档，提供版本管理、成员协作、媒体资源管理等功能。

### 1.3 技术栈选型

#### 核心框架
- **Vue 3** - 渐进式 JavaScript 框架（Composition API）
- **TypeScript** - 类型安全的 JavaScript 超集
- **Vite** - 快速的前端构建工具

#### UI 组件库
- **Element Plus** - Vue 3 组件库
- **Pinia** - Vue 状态管理

#### 富文本编辑器
- **ProseMirror** 或 **TipTap** - 支持协同编辑的富文本编辑器框架
- **Yjs** - CRDT（无冲突复制数据类型）库，用于实现实时协作

#### WebSocket 通信
- **原生 WebSocket API** 或 **Socket.io-client**
- 自定义消息协议处理

#### HTTP 请求
- **Axios** - Promise based HTTP client

#### 其他工具库
- **Day.js** - 日期时间处理
- **Lodash-es** - 工具函数库
- **Vue Router** - 路由管理

## 四、核心技术实现方案

### 4.1 WebSocket 连接管理

```typescript
// utils/websocket.ts
class WebSocketManager {
  private ws: WebSocket | null = null;
  private reconnectAttempts = 0;
  private maxReconnectAttempts = 5;
  private messageHandlers: Map<string, Function> = new Map();

  connect(token: string) {
    const wsUrl = `ws://localhost:8080/api/ws?token=${token}`;
    this.ws = new WebSocket(wsUrl);
    
    this.ws.onopen = () => {
      console.log('WebSocket connected');
      this.reconnectAttempts = 0;
      this.startHeartbeat();
    };
    
    this.ws.onmessage = (event) => {
      const message = JSON.parse(event.data);
      this.handleMessage(message);
    };
    
    this.ws.onclose = () => {
      console.log('WebSocket closed');
      this.attemptReconnect(token);
    };
    
    this.ws.onerror = (error) => {
      console.error('WebSocket error:', error);
    };
  }

  send(message: WebSocketMessage) {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(message));
    }
  }

  on(type: string, handler: Function) {
    this.messageHandlers.set(type, handler);
  }

  private handleMessage(message: WebSocketMessage) {
    const handler = this.messageHandlers.get(message.type);
    if (handler) {
      handler(message);
    }
  }

  private startHeartbeat() {
    setInterval(() => {
      this.send({ type: 'HEARTBEAT', timestamp: Date.now() });
    }, 30000);
  }

  private attemptReconnect(token: string) {
    if (this.reconnectAttempts < this.maxReconnectAttempts) {
      this.reconnectAttempts++;
      setTimeout(() => this.connect(token), 3000 * this.reconnectAttempts);
    }
  }

  disconnect() {
    if (this.ws) {
      this.ws.close();
      this.ws = null;
    }
  }
}
```

### 4.3 Axios 请求拦截器

```typescript
// utils/request.ts
import axios from 'axios';
import { useAuthStore } from '@/stores/auth';

const request = axios.create({
  baseURL: '/api',
  timeout: 30000
});

// 请求拦截器
request.interceptors.request.use(
  (config) => {
    const authStore = useAuthStore();
    if (authStore.token) {
      config.headers.Authorization = `Bearer ${authStore.token}`;
    }
    return config;
  },
  (error) => {
    return Promise.reject(error);
  }
);

// 响应拦截器
request.interceptors.response.use(
  (response) => {
    return response.data;
  },
  async (error) => {
    const authStore = useAuthStore();
    
    if (error.response?.status === 401) {
      // Token 过期，尝试刷新
      try {
        await authStore.refreshToken();
        // 重试原请求
        return request(error.config);
      } catch (e) {
        // 刷新失败，跳转登录页
        authStore.logout();
        window.location.href = '/login';
      }
    }
    
    return Promise.reject(error);
  }
);

export default request;
```

### 4.4 WebSocket 消息处理规范

**⚠️ 关键：消息类型必须使用小写下划线格式**

```typescript
// utils/websocket.ts
class WebSocketClient {
  private ws: WebSocket | null = null;
  private messageHandlers: Map<string, Function> = new Map();

  /**
   * 注册消息处理器
   * @param type 消息类型（必须是小写下划线格式）
   * @param handler 处理函数
   */
  on(type: string, handler: (data: any) => void) {
    // ✅ 正确：使用小写匹配
    this.messageHandlers.set(type.toLowerCase(), handler);
  }

  send(message: any) {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      // ✅ 确保 type 是小写下划线格式
      if (message.type) {
        message.type = message.type.toLowerCase();
      }
      this.ws.send(JSON.stringify(message));
    }
  }

  private handleMessage(message: any) {
    // ✅ 使用小写匹配
    const handler = this.messageHandlers.get(message.type.toLowerCase());
    if (handler) {
      handler(message.data);
    } else {
      console.warn('⚠️ 未注册的消息处理器:', message.type);
    }
  }
}

export default new WebSocketClient();
```

| 场景 | ❌ 错误写法 | ✅ 正确写法 |

| 监听操作广播 | `wsClient.on('OPERATION_BROADCAST', ...)` | `wsClient.on('operation_broadcast', ...)` |

### 4.5 文件分块上传

```typescript
// composables/useUpload.ts
import { ref } from 'vue';
import { mediaApi } from '@/api/media';

export function useChunkUpload() {
  const progress = ref(0);
  const isUploading = ref(false);

  const CHUNK_SIZE = 5 * 1024 * 1024; // 5MB

  async function uploadFile(file: File, docId: number) {
    isUploading.value = true;
    progress.value = 0;

    try {
      // 1. 初始化上传
      const initResponse = await mediaApi.initChunkUpload({
        filename: file.name,
        fileSize: file.size,
        mediaType: file.type,
        chunkCount: Math.ceil(file.size / CHUNK_SIZE)
      });

      const uploadId = initResponse.uploadId;

      // 2. 分块上传
      const chunks = Math.ceil(file.size / CHUNK_SIZE);
      for (let i = 0; i < chunks; i++) {
        const start = i * CHUNK_SIZE;
        const end = Math.min(start + CHUNK_SIZE, file.size);
        const chunk = file.slice(start, end);

        await mediaApi.uploadChunk({
          uploadId,
          chunkIndex: i,
          chunk
        });

        progress.value = Math.round(((i + 1) / chunks) * 100);
      }

      // 3. 完成上传
      const result = await mediaApi.completeUpload({
        uploadId,
        filename: file.name,
        fileSize: file.size,
        mediaType: file.type
      });

      return result;
    } catch (error) {
      console.error('Upload failed:', error);
      throw error;
    } finally {
      isUploading.value = false;
    }
  }

  return {
    progress,
    isUploading,
    uploadFile
  };
}
```

## 七、关键技术挑战与解决方案

### 7.1 实时协作冲突处理
**挑战**: 多用户同时编辑可能导致内容冲突

**解决方案**:
- 使用 Yjs CRDT 库处理并发编辑
- 实现操作转换（OT）算法作为备选方案
- 服务端进行最终一致性校验

### 7.2 大文件上传
**挑战**: 大文件上传可能超时或失败

**解决方案**:
- 实现分块上传机制
- 支持断点续传
- 显示上传进度
- 提供取消上传功能

### 7.3 WebSocket 连接稳定性
**挑战**: 网络波动导致连接断开

**解决方案**:
- 实现自动重连机制（指数退避）
- 心跳检测保持连接活跃
- 连接断开时缓存本地操作
- 重连后进行增量同步

### 7.4 性能优化
**挑战**: 大量实时消息可能导致性能问题

**解决方案**:
- 消息节流和防抖
- 虚拟滚动处理长列表
- 懒加载和代码分割
- 使用 Web Worker 处理复杂计算

### 7.5 离线支持
**挑战**: 网络中断时的编辑体验

**解决方案**:
- 使用 IndexedDB 缓存文档内容
- 本地操作队列
- 网络恢复后自动同步
- 冲突检测和解决提示

## 八、安全性考虑

### 8.1 认证与授权
- JWT Token 存储在 HttpOnly Cookie 或内存中
- Token 自动刷新机制
- 路由级别权限控制
- API 请求携带 Authorization header

### 8.2 XSS 防护
- 对用户输入进行转义
- 使用 CSP (Content Security Policy)
- 避免使用 innerHTML

### 8.3 CSRF 防护
- 使用 SameSite Cookie 属性
- 关键操作需要二次确认

### 8.4 WebSocket 安全
- WebSocket 握手时验证 Token
- 消息级别权限校验
- 防止消息注入
